pywiktionary.parsers.english_parser module¶

class pywiktionary.parsers.english_parser.EnglishParser(html)¶

Bases: pywiktionary.parsers.basic_parser.BasicParser

Parser for english wiktionary pages

get_meanings(meaning_types=['noun', 'verb', 'adverb', 'adjective', 'preposition', 'numeral', 'proper_noun', 'pronoun', 'particle', 'interjection', 'phrase', 'contraction', 'conjunction', 'article', 'suffix', 'abbreviation', 'initialism', 'proverb', 'prefix'], get_examples=True, get_empty_meaning_types=False)¶

Get english meanings of the word find in the Wiktionary HTML page contained in this parser

Parameters

  • meaning_types (list) – List containing meaning type keys to looking for. By default this list contain all foundable keys

  • get_examples (boolean) – Specify if the result should contain example of meanings. By default this is True

  • get_empty_meaning_types (boolean) – Specify if empty meaning type keys should be included in the returned wiktionary. By default this is False

Return dictionary

Return dictionary with all found meanings divided by meaning type keys. Each entry contain a list of dictionaries with two entry: ‘meaning’ which is a meaning of the word, and ‘example’ with a list of examples, if are present and should be returned, for the retrieved meaning.

parse()¶

Parse the page to return word attributes. This is an extension of BasicParser

Returns

Word attributes. With first release this contain just the ‘meaning’ key

Raises

ValueError – Raise a ValueError exception if the parser has an empty invalid html
